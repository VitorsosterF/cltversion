Gerenciamento de Configuração de Software

Controle de Versão

É uma prática extremamente importante no Desenvolvimento de Software. Pois ela gerencia as alterações realizadas nos arquivos, como códigos-fontes e outros, permitindo assim o rastreamento dessas ações feitas. Isso possibilita a colaboração entre desenvolvedores de maneira mais eficaz, mantendo um histórico das versões anteriores, e também sendo possível reverter para as mesmas.

A reversão para versões anteriores aconteceria em caso de alguma mudança ter sido feita incorretamente, ou para recuperar algo que foi perdido. Dito isso, a ferramente mais usada para realizar esta prática é o GIT, por conta de sua eficácia e versatilidade ao trabalhar em equipe.

Controle de Mudanças

Diferente da anterior, o controle de mudanças consiste mais na documentação das mudanças realizadas no projeto, sendo feitas avaliações e aprovações antes de cada alteração. Isso tudo é feito a fim de garantir que todas as modificações que forem implementadas no projeto aconteçam de forma organizada, assim evitando confusões futuras.

Além disso, está prática abrange o projeto como um todo, não se limitando apenas ao código. Ela está presente na tomada de decisões desde a proposta até a implementação.

Integração Contínua

Integração contínua é uma outra prática na área de desenvolvimento de software que é basicamente a integração do código de poucos em poucos, fazendo pequenas alterações frequentes para assim haver uma detecção de erros mais precisa.

Isso faz com que caso haja algum erro, ele tenha menos impacto no código e no projeto como um todo, pois essa prática permite a rápida identificação dele.
